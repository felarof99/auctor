# Auctor JSON Results & Dashboard — Amendment

Date: 2026-04-17
Status: Approved for planning
Author: nithin@browseros.com
Amends: `docs/superpowers/specs/2026-04-17-auctor-bundle-configs-microscope-design.md`

## Summary

Amend the approved bundle spec with three changes:

1. **Per-bundle directory layout.** Committed config is `configs/<bundle>/<bundle>_config.yaml`. Results live in `configs/<bundle>/.results/` (gitignored).
2. **Remove Convex from the CLI.** The `auctor analyze` and `auctor configure` pipelines no longer call `convex-client`. The CLI becomes a pure producer of JSON files. Convex returns later as a separate `auctor push` step operating on the JSON outputs.
3. **One JSON file per repo inside a bundle.** No CLI-side aggregate file. A shared aggregation module computes the bundle-level view at read time; the dashboard uses it for display, and the CLI uses it for its stdout leaderboard.

Scope preserved from the parent spec: YAML schema fields (`name`, `server_url`, `repos`, `engineers`), the three commands (`configure` / `analyze` / `microscope`), and the classify→score pipeline. `convex_url` remains in the schema but is unused for now.

## Motivation

- Fast iteration on the JSON report shape without DB round-trips or schema migrations.
- The dashboard becomes a dev-only renderer of local files — no Convex read path — so we can change the shape freely.
- Per-bundle folders keep config + generated output adjacent, which matches the mental model of "one bundle = one workspace".

## YAML path convention

`loadBundle(path)` accepts `configs/<bundle>/<bundle>_config.yaml` and:
- Parses YAML.
- Asserts `basename(dirname(path)) === config.name`. Mismatch → error at load time.
- Returns a `BundleConfig` with `configDir = dirname(path)` attached, used by writers to resolve `<configDir>/.results/...`.

### New analyze flow

Signature: `analyze(configPath: string, timeWindow: string, jsonPath?: string): Promise<void>`

```
loadBundle(configPath)
  → validate bundle.repos[].path exists and is a git repo; skip missing with warning
  → parseTimeWindow(timeWindow)
  → for each repo in bundle.repos (sequential):
      analyzeSingleRepo(repo, bundle, since, daysInWindow) → PerRepoResult
      write configs/<bundle>/.results/<repo.name>.json
  → aggregateBundle(perRepoResults) via shared package
  → print combined leaderboard + sparklines to stdout
  → if --json <file> passed: write a RepoReport-shaped file with repo=bundle.name, authors=aggregate
```

`analyzeSingleRepo`:

```
getGitLog + getMergeCommits
  → filter by bundle.engineers
  → extractBranchDayUnits + extractPrUnits
  → getDiffForCommits (hydrate)
  → classifyWorkUnits(server_url) OR default classification if server_url missing
  → calculateUnitScore per unit
  → calculateAuthorScore per author
  → computeDailyScores per author
  → return PerRepoResult { bundle, repo, window, window_days, generated_at, authors[] }
```

Zero Convex calls. Every run is a full re-classification against the server; the server's own cache (unchanged) covers cost.

### Microscope output

Unchanged from parent spec except the path is `configs/<bundle>/.results/<bundle>-microscope-<username>-<YYYYMMDD-HHMM>.json`. Dashboard ignores these files for now.

### CLI stdout

`analyze` still prints one aggregated leaderboard + sparklines (via the shared aggregate module), matching the parent spec's UX. No per-repo leaderboards printed.

### Error handling

- Repo path missing or not a git repo → warn, skip, continue. All repos skipped → exit 1.
- Classification server unreachable with `server_url` set → warn and fall back to default classification for that repo.
- No engineers matched in any repo → exit 0 with a clear message.

## Shared aggregation module

Location: `packages/shared/src/aggregate.ts`.

Exports:
- `type BundleAuthorStats` — same fields as `RepoAuthorStats` plus `repos: string[]` listing which repos the author contributed to.
- `type BundleAggregate { bundle: string; window_days: number; generated_at: string; authors: BundleAuthorStats[] }`.
- `function aggregateBundle(reports: RepoReport[]): BundleAggregate`.

Semantics:
- Group authors by username across reports.
- Sum `commits`, `prs`, `insertions`, `deletions`, `net`, `score`.
- Merge `daily_scores` by index (all per-repo arrays are the same length for a given window). Result is element-wise sum.
- `generated_at` = max of input reports.
- `window_days` = assert all inputs equal; error otherwise.

Consumers:
- `apps/cli/src/commands/analyze.ts` — for the stdout leaderboard.
- `apps/dashboard/src/hooks/use-reports.ts` — for the bundle aggregate tab.

Tests: `packages/shared/src/aggregate.test.ts` with a two-report fixture.

## Dashboard changes

### Sync script

Location: `apps/dashboard/scripts/sync-data.ts`.

Steps:
1. Glob `configs/*/.results/*.json` from the repo root (resolve via `import.meta.dir` walking up).
2. Skip files matching `*-microscope-*.json`.
3. For each `configs/<bundle>/.results/<repo>.json`, copy to `apps/dashboard/public/data/<bundle>__<repo>.json`.
4. Write `apps/dashboard/public/data/manifest.json` with shape `{ bundles: [{ name, repos: string[] }] }`.
5. Delete any files in `public/data/` that no longer have a source (except `manifest.json`).

Package scripts (root `package.json`):
```json
"dashboard:sync": "bun apps/dashboard/scripts/sync-data.ts",
"dashboard:dev": "bun run dashboard:sync && bun --cwd apps/dashboard run dev"
```

### `use-reports.ts` rewrite

- Manifest shape changes from `string[]` to `{ bundles: [{ name, repos }] }`.
- Hook returns `{ bundles: Record<string, { repos: Record<string, RepoReport>, aggregate: BundleAggregate }> }`.
- Fetches each per-repo file using the `<bundle>__<repo>.json` filename.
- Computes `aggregate` via `aggregateBundle` from `@auctor/shared/aggregate`.

### `App.tsx` tab structure

- Primary tabs: one per bundle.
- Secondary tabs inside a bundle: `"Aggregate"` (default) + one per repo.
- Aggregate tab renders `BundleAggregate.authors` using the existing table markup.
- Per-repo tab renders `RepoReport.authors` unchanged.

### `@auctor/shared/report.ts`

- Add `BundleAggregate`, `BundleAuthorStats` re-exports (or move them here if `aggregate.ts` lives alongside).
- Keep `RepoReport` backward-compatible; add optional `bundle?: string` field.

## Rollout

Single PR on `main`. No feature flag. Breaking for the single user of this tool.

1. Land spec, code, tests, and `.gitignore` change in one series of commits.
2. Delete `apps/dashboard/public/data/sample-repo.json` and `competitor-repo.json`.
3. Create `configs/browseros/browseros_config.yaml` manually or via a one-shot `auctor configure` invocation so the repo ships with one real bundle.

   - `bun run analyze configs/browseros/browseros_config.yaml 14d` → per-repo JSONs land in `configs/browseros/.results/`.

## Non-goals

- No Convex write path from the CLI. Re-introduction will be a separate spec for `auctor push`.
- No dashboard deploy. Local dev only until `auctor push` exists.
- No microscope rendering in the dashboard. CLI-only for now.
- No backward compatibility with `.auctor.json` or with the old `configs/<bundle>.yaml` flat layout (neither exists in the committed repo).
